PMI and CAM Integration in SIEMENS NX

A GWiEIT 2025 Summer School Project. PMI-CAM Integration in Siemens NX

What is PMI and CAM?

PMI

It stands for Product Manufacture Information and it refers to the manufacturing information embedded within a 3D CAD model. It can be anything from the depth of a hole, to the thickness of a wall, for example.

CAM

It stands for Computer Aided Manufacturing and is the software used to translate the PMIs into instructions for automated manufacturing equipment like CNC machines. It is a set of instructions from which a digital part is manufactured in the real world. Its main goal is to speed up production and create precise, consistent components and tools, often using only the needed material to reduce waste and energy use.

Context

The Problem

Currently, there is no connection shown between PMI and CAM operations in NX. When a user adds CAM operations, they need to check manually if the relevant PMI is matching, toggling between the two components. Poor interoperability between digital tools, which is often worsened by manual data entry, costs the U.S. capital facilities industry about $15.8 billion per year.

The Solution

To solve the problem, we created an integrated dialog window inside Siemens NX, targeted toward CAM engineers and programmers, that shows the connection between PMI and CAM operations and allows the user to highlight them on the part that will be manufactured. This can help save time and resources when manufacturing products.

The Innovation

When engineering teams have to devote time and focus to ensuring data consistency across systems, innovation is equally limited. With a digital manufacture market that is predicted to reach USD 1,813.07 billion by 2032, this solution has the potential to enhance production efficiency, reduce operating costs and improve product quality.

Key Features

Overview

The PMI–CAM dialog is a custom-built tool developed for Siemens NX that allows users to easily explore how PMI annotations are linked to CAM operations within a part file. It is specifically designed to assist engineers and NX programmers during the manufacturing setup process by clearly showing which product features are referenced by each CAM operation. This tool has been tested and validated using Siemens NX 2412. By integrating design and manufacturing data in one interface, the dialog reduces the need to switch between windows and improves clarity. It also helps ensure that critical tolerances, finishes, and dimensions are not overlooked during machining, supporting more accurate and efficient workflows.

Integrated Dialog Window

Building on the work that was done by the GWiEIT cohort of Track 3, we were able to fully integrate the dialog window inside NX using the UI Blockstyler, a user interface tool in Siemens NX that allows developers to create custom dialog windows. The final dialog box, which can be opened with a button, has each PMI and CAM listed in their respective list boxes.

Synchronous PMI-CAM Highlighting

Both PMI and CAM list boxes have checkboxes on the left side. When a PMI is clicked, the checkbox gets crossed off, and so do all the respective CAM operation in the list box below. Furthermore, there is a tree list box at the top of the dialog window, where all the selected PMI with their respective CAM operations can be seen displayed next to each other. This helps visually organise the screen when a part has several PMI and CAM operations.

Reverse Mapping

In addition to being able to click the PMI and seeing all the CAM operations associated to a surface/feature, the user can also select a specific CAM operation and all the related PMIs will be highlighted. Additionally, the program alerts the user with a notification if no PMI or CAM operations are found in the part file.
